# 翻译代理系统Web界面使用说明

## 改进方案概述

为翻译代理系统添加了基于Streamlit的Web界面，主要改进如下：

1. **创建独立的Streamlit应用文件**：`streamlit_app.py`，包含完整的Web界面实现
2. **参数可视化配置**：将所有从`config.ini`文件读取的参数通过下拉列表、文本框等控件在Web界面中呈现
3. **临时配置文件生成**：根据用户在Web界面中输入的参数，动态生成临时配置文件供翻译代理使用
4. **执行流程可视化**：添加进度条和状态文本，实时显示翻译任务执行进度
5. **结果展示优化**：通过Streamlit的UI组件美观地展示翻译结果
6. **依赖管理**：在`requirements.txt`中添加了Streamlit依赖

## 主要功能

Web界面提供了以下功能：

### 1. GitHub配置
- GitHub访问令牌输入（密码类型，保护用户敏感信息）
- GitHub仓库地址输入（支持完整URL或owner/repo格式）
- 分支选择（支持多个分支，用逗号分隔）

### 2. 翻译配置
- 目标语言选择（下拉列表：简体中文、繁体中文、英文、日语）
- 专业术语词典文件路径输入
- 最大文件大小限制设置（单位：KB）

### 3. LLM配置
- OpenAI API密钥输入（密码类型）
- 通义千问API密钥输入（密码类型）
- LLM模型选择（下拉列表：gpt-3.5-turbo、gpt-4、qwen-plus、qwen-max）

### 4. 目录配置
- 下载文件保存目录设置
- 翻译结果保存目录设置
- 临时文件目录设置

### 5. 执行与结果展示
- 一键执行翻译按钮
- 实时进度条显示
- 详细的翻译结果展示
- 执行状态和错误提示

## 安装和使用方法

### 1. 安装依赖

首先，确保已安装所有必要的依赖包：

```bash
pip install -r requirements.txt
```

### 2. 启动Web界面

在项目根目录下执行以下命令启动Streamlit Web界面：

```bash
streamlit run streamlit_app.py
```

### 3. 使用步骤

1. 在浏览器中打开Streamlit界面（通常是 http://localhost:8501）
2. 填写所有必要的配置参数（至少需要填写GitHub仓库地址）
3. 点击"执行翻译"按钮开始翻译流程
4. 查看实时进度和最终结果

## 注意事项

1. Web界面中的敏感信息（如API密钥）以密码类型输入，不会在界面上明文显示
2. 翻译大文件时可能需要较长时间，请耐心等待
3. 执行过程中会生成临时配置文件，执行完成后会自动清理
4. 如需修改翻译代理的核心逻辑，请修改`main_optimized.py`文件

## 技术细节

1. **Streamlit应用结构**：
   - 使用表单组织所有配置项
   - 采用列布局优化界面空间使用
   - 使用侧边栏提供使用说明

2. **与翻译代理的集成方式**：
   - 通过动态生成临时配置文件实现参数传递
   - 保留了原有的命令行功能，实现双模式运行

3. **用户体验优化**：
   - 添加详细的帮助文本和提示信息
   - 提供直观的进度反馈
   - 结果展示支持展开/折叠，方便查看大量结果

## 故障排除

1. 如果遇到"找不到模块"的错误，请检查Python路径是否正确
2. 如果GitHub连接失败，请检查仓库地址和访问令牌是否正确
3. 如果翻译结果不理想，请尝试提供更准确的API密钥和适合的模型
4. 如果界面显示异常，请确保Streamlit版本为最新版

---

通过这个Web界面，用户可以更直观、更便捷地配置和使用翻译代理系统，无需手动编辑配置文件，大大提高了工作效率。
